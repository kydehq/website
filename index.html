↳ FAQ

Frequently asked questions

What counts as an agent?

Any software component that makes LLM API calls — chatbots, autonomous coding agents, AI assistants, workflow automation bots. Each unique agent identity (agent_id) in your infrastructure counts as one governed agent.

Do you store our prompts and responses?

KYDE stores cryptographic hashes (SHA-256) of inputs and outputs, not the raw content. The causal context (last 5 messages) is stored locally in your append-only database for audit purposes. Your data never leaves your infrastructure.

Is this GDPR compliant?

Yes. KYDE runs as a local proxy within your infrastructure. No data is transmitted to external servers. The database stores hashes and metadata, not personal data. You maintain full data sovereignty.

What happens if the proxy goes down?

KYDE WITNESS is designed as a transparent proxy. In case of failure, you can configure fail-open (agents continue without logging) or fail-closed (agents are blocked until the proxy recovers). Enterprise deployments run redundant instances.

Can I export my audit data?

Yes. Use the CLI tool: witness export --format json. Exports include full chain integrity metadata, allowing independent verification by any third party or regulator.

How does this differ from Zenity or Lakera?

Zenity and Lakera protect prompts (semantic security). KYDE governs agents (business logic). They inspect inputs/outputs. We inspect authority — what an agent is allowed to do, under what policy, with cryptographic proof. Different layers, complementary solutions.

What if we already use OpenAI's built-in logging?

Vendor-provided logs are self-reported and provider-specific. A compromised system can alter them. KYDE provides provider-agnostic, cryptographically signed logs that are independently verifiable. One governance layer across all your LLM providers.

Do we need developer involvement to deploy?

No. IT can deploy KYDE WITNESS via a single environment variable or Group Policy push. Agents that already use OpenAI-compatible endpoints require zero code changes. Typical deployment takes under 5 minutes.
